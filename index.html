Nebula: Character Management Software

Nebula is library for maintaining characters for RPGs. It is designed to be fairly system independent, although like all software only time will tell how successful it will be at that.

It is heavily weighted towards the mechanics side of the design, especially number heavy ones. The first systems targetted are

  • Rolemaster (RMSS/FRP at first)
  • Harp
  • DND 3.5

Once these systems are complete (or at least usable) additional focus will be on:

  • Dogs in the Vineyard
  • My life with master
  • World of Darkness games (Mage, Vampire, Werewolf et al)

Other requested systems:

  • Call of Cthulhu
  • Houses of the Blooded
  • Other Rolemaster flavours

Components

There are currently 4 main components to Nebula.

LibNebula

The main library for nebula. It stores and manages the data at runtime for the characters.

Eta Carinae

A EFL based viewer for nebula characters.

Templates

System templates and data

Protocol and standards

Protocol and standards covers the general rules which allow an arbitrary application to render an arbitrary character in an arbitrary system in a useful manner.

Status

Current status is:

LibNebula

Mostly complete. The core of the library is complete. Work needs to be done on supporting more advanced features such as character auditing and rich display. (So you can see exactly where that +27 strength bonus comes from).

The code is fairly well documented.

Eta Carinae

Only basic viewer, no editing at all. Minimal documentation.

Templates

Proof of concept templates for Harp and DnD3.5.

Protocol

Basic protocol done in Eta Carinae, and templates. Not documented
